# ASH-TCU-K4A — Shape Sweep 2x1 To 1x2 / Grid Orientation Parity Seal

## Status

`SPEC_COMMITTED_ASH_TCU_K4A_SHAPE_SWEEP_2X1_TO_1X2_GRID_ORIENTATION_PARITY_SEAL`

## Scope

ASH-TCU-K4A verifies that the K3B descriptor indexing path and K3C/K3D multi-macro machinery survive a grid orientation flip from horizontal `2x1` to vertical `1x2`.

K3D proved native `2x1` multi-macro dispatch correctness with `dispatch_workgroups=[2,1,1]`, 2048-byte `ReadbackBufferBytes`, and 512-element CPU oracle comparison. K4A uses the same descriptor-indexing formula, but changes grid orientation to `macro_grid_width=1`, `macro_grid_height=2`, and `dispatch_workgroups=[1,2,1]`.

K4A does not open production multi-macro inference. K4A does not make performance claims. K4A does not reuse K3D readback as primary evidence.

## Orientation parity target

K3D used horizontal mapping:

```text
2x1:
workgroup_id [0,0,0] -> macro_index 0
workgroup_id [1,0,0] -> macro_index 1
```

K4A must use vertical mapping with the same formula:

```text
macro_index = workgroup_id.y * macro_grid_width + workgroup_id.x
```

For 1x2:

```text
workgroup_id [0,0,0] -> macro_index 0
workgroup_id [0,1,0] -> macro_index 1
```

This proves the descriptor indexing path is not accidentally hardwired to horizontal x-axis dispatch.
