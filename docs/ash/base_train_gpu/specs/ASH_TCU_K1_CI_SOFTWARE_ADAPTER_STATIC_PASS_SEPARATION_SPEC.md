# ASH-TCU-K1 — CI Software Adapter / Static PASS Separation Seal

## Status

`SPEC_COMMITTED_ASH_TCU_K1_CI_SOFTWARE_ADAPTER_STATIC_PASS_SEPARATION_SEAL`

## Scope

ASH-TCU-K1 separates static PASS, compile PASS, unit-test PASS, software-adapter smoke PASS, NotRun, and Fail states before CPU oracle/readback or multi-macro widening work proceeds.

This spec is committed to GitHub as documentation only. Runtime receipts and manifests are not committed by this spec and must be generated by local Rust or CI execution.

## Rationale

The ASH bake environment has repeatedly allowed static file/surface checks to look like execution evidence. K1 closes that gap by forcing status separation. A tree may pass static checks, but it may not claim executed PASS unless cargo build/tests and, when available, software-adapter smoke execution are recorded.

K1 is not a performance patch. K1 is not a TensorCube math patch. K1 does not widen dispatch. It prepares the execution-state hygiene required by ASH-TCU-K2 and ASH-TCU-K3.

## Status model

```text
StaticPass
CargoUnavailable
CargoBuildPass
CargoBuildFail
UnitTestPass
UnitTestFail
SoftwareAdapterUnavailable
SoftwareAdapterSmokePass
SoftwareAdapterSmokeFail
NativeGpuNotRequired
```

## Status rules

```text
StaticPass is not ExecutedPass.
CargoUnavailable is NotRun, not PASS and not FAIL.
SoftwareAdapterUnavailable is NotRun, not PASS and not FAIL.
SoftwareAdapterSmokePass is executed adapter evidence, not performance evidence.
NativeGpuNotRequired remains neutral for K1.
```

## Software adapter smoke contract

The K1 software adapter smoke is intentionally tiny:

```text
input u32 buffer = 41
minimal compute shader adds 1
readback expects 42
```

This smoke proves adapter/device/shader/dispatch/readback wiring only. It does not prove TensorCube correctness, CPU oracle equivalence, or performance.

```text

2. StaticPass and ExecutedPass are structurally separated.
3. CargoUnavailable is NotRun, not PASS and not FAIL.
4. SoftwareAdapterUnavailable is NotRun, not PASS and not FAIL.
5. Software adapter smoke harness exists.
6. K1 local audit binary exists.
7. K1 local audit binary writes runtime JSON outputs.
8. Runtime JSON outputs are not pre-baked into the ZIP as evidence.

10. Production dispatch remains closed.
11. Performance claim remains forbidden.
```
