# ASH-TCU-K2 — CPU Oracle + Mapped Readback Digest Harness Seal

## Status

`SPEC_COMMITTED_ASH_TCU_K2_CPU_ORACLE_MAPPED_READBACK_DIGEST_HARNESS_SEAL`

## Scope

ASH-TCU-K2 establishes CPU oracle, mapped readback digest, and tolerance comparison as the TensorCube correctness harness before any multi-macro widening or valid benchmark work proceeds.

This spec is committed to GitHub as documentation only. Runtime receipts and manifests are not committed by this spec and must be generated by local Rust or CI execution.

## Rationale

K0 sealed kernel identity and SSOT. K1 separated static PASS from executed PASS. K2 adds the correctness layer: fixed fixture, CPU oracle, mapped ReadbackBufferBytes digest, materialized tensor cross-check policy, and abs/rel tolerance comparison.

K2 is not a performance patch. K2 is not a TensorCube math patch. K2 does not widen dispatch. It prepares the correctness evidence required by ASH-TCU-K3A and later valid benchmark work.

## Correctness contract

```text
fixed seed fixture
→ CPU oracle execute
→ GPU dispatch
→ copy GPU output buffer
→ map_async + device poll
→ mapped range bytes acquire
→ SHA-256 digest over exact logical bytes
→ unpack GPU output
→ abs/rel tolerance compare with CPU oracle
→ receipt write
```

## Evidence policy

```text
primary_gpu_digest_source = ReadbackBufferBytes
materialized_tensor_digest_role = cross_check_only
materialized_tensor_fallback_allowed = false
```

If mapped readback bytes are unavailable, K2 must record NotRun or Blocked. It must not claim correctness PASS.
