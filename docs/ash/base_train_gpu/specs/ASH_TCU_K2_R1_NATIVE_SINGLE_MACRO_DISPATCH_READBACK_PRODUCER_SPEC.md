# ASH-TCU-K2-R1 — Native Single-Macro Dispatch Readback Producer / CPU Oracle Fixture To GPU Readback Bytes Seal

## Status

`SPEC_COMMITTED_ASH_TCU_K2_R1_NATIVE_SINGLE_MACRO_DISPATCH_READBACK_PRODUCER_SEAL`

## Scope

ASH-TCU-K2-R1 produces the mapped ReadbackBufferBytes required by ASH-TCU-K2. K2 already provides the CPU oracle and correctness verifier. K2-R1 adds the producer that runs a native single-macro TensorCube dispatch against the K2 fixture and writes raw f32 little-endian GPU output bytes.

This spec is committed to GitHub as documentation only. Runtime receipts, readback bytes, and manifests are not committed by this spec and must be generated by local Rust execution.

## Rationale

Existing R10-R3 readback JSON receipts cannot satisfy K2 because they record no actual mapped bytes. The observed state includes `gpu_output_buffer_present=false`, `readback_buffer_created=false`, `copy_to_readback_encoded=false`, `copy_to_readback_submitted=false`, `map_async_completed=false`, `mapped_range_acquired=false`, `mapped_range_byte_len=0`, and `output_bytes_captured=false`.

K2-R1 therefore does not extract bytes from existing JSON receipts. It performs a fresh native dispatch and maps the readback buffer.

## Included changes

```text

- Reuse K2 CPU oracle
- Add native single-macro dispatch readback producer
- Create GPU input, output, control, and readback buffers
- Encode dispatch_workgroups(1, 1, 1)
- Encode copy_buffer_to_buffer(output -> readback)
- Submit queue
- Request map_async(Read)
- Poll device
- Acquire mapped range bytes
- Write raw f32 little-endian .bin file
- Generate ReadbackBufferBytes digest receipt
- Compare readback f32 values with K2 CPU oracle
- Generate K2 correctness bridge receipt
```
