# ASH-TCU-K3D — Native 2x1 Multi-Macro Dispatch Readback / Grid Plan To GPU Output Correctness Seal

## Status

`SPEC_COMMITTED_ASH_TCU_K3D_NATIVE_2X1_MULTI_MACRO_DISPATCH_READBACK_CORRECTNESS_SEAL`

## Scope

ASH-TCU-K3D consumes the K3C `MacroGridPlan` and executes the first native 2x1 multi-macro TensorCube dispatch. The adoption evidence is a new 2048-byte `ReadbackBufferBytes` file produced by K3D and compared against a 512-element CPU oracle.

K3D does not open production multi-macro inference. K3D does not make performance claims. K3D does not reuse K2-R1 single-macro readback as primary evidence.

## Primary evidence rule

K3D primary GPU evidence must be:

```text
ReadbackBufferBytes generated by ASH-TCU-K3D native 2x1 dispatch
byte_len = 2048
scalar_count = 512
format = raw f32 little-endian
```

Forbidden as primary evidence:

```text
K2-R1 readback bytes
K2-R1 digest
K2-R1 readback copied twice
CPU oracle bytes
MaterializedTensorBytes fallback
JSON debug arrays
base64 wrappers
empty bytes
padding bytes as logical output
```

K2-R1 is allowed only as a prior gate.
