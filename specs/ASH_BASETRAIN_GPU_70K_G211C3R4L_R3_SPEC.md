# G211C3R4L-R3
## Benchmark Target Alignment Repair / Burn Matmul Versus Logical16 SoftMatrix Candidate Seal

## Scope

G211C3R4L-R3 aligns the benchmark targets for the QW-55A-0-R3 Logical16 KPanel candidate and the real Burn WebGPU matmul baseline.

This patch does not declare a speed winner. It only proves that both paths can be compared under the same input SSOT, same logical shape, same dtype, same native adapter/device setup, and same readback/correctness policy.

## Parent seal

```text
parent_patch_id = QW-55A-0-R3
parent_verdict = PASS_QW55A0_R3_LOGICAL16_GENERAL_K_PANEL_ACCUMULATION_SEALED
candidate_kernel_kind = Logical16FromTile8SoftGroupKPanel
```

## Required targets

```text
baseline_target_kind = BurnWebGpuMatmul
candidate_target_kind = Logical16FromTile8SoftGroupKPanel
```

Forbidden baseline targets:

```text
InlineNaiveShader
HandWrittenToyShader
CpuReference
MockBurn
StaticManifestOnly
```

## Shape fixtures

```text
k_panel_count = 1 -> M=16 N=16 K=8
k_panel_count = 2 -> M=16 N=16 K=16
k_panel_count = 4 -> M=16 N=16 K=32
k_panel_count = 8 -> M=16 N=16 K=64
```

## Input SSOT

Each fixture is generated once as logical A/B values. Burn tensors and Logical16 8x8 candidate tiles are both derived from that same scalar source.

```text
same_input_ssot_used = true
cpu_reference_used_for_correctness_only = true
cpu_reference_used_as_benchmark_baseline = false
```

## Same-device alignment

The audit creates one native WGPU adapter/device/queue setup and initializes Burn WGPU from that setup. The Logical16 candidate dispatch uses the raw device/queue from the same setup.

```text
same_adapter_required = true
same_backend_required = true
adapter_fingerprint_present = true
```

## Correctness comparisons

Each fixture must compare:

```text
Burn vs CPU reference
Candidate vs CPU reference
Candidate vs Burn
```

Initial tolerance:

```text
abs_tolerance = 1e-4
rel_tolerance = 1e-4
```

## Timing policy

Timing is recorded but not promoted into a speed winner.

```text
warmup_iterations = 3
measure_iterations = 10
speedup_required_for_pass = false
performance_winner_declared = false
```

```bash
cargo run -p orchestrator_local --features orchestrator_tcu_audit_bins --bin g211c3r4l_r3_benchmark_target_alignment_audit
```

```bash

```

With fixtures:


## PASS verdict

```text
PASS_G211C3R4L_R3_BENCHMARK_TARGET_ALIGNMENT_SEALED
```

PASS means the comparison targets are aligned. It does not mean the Logical16 candidate is faster than Burn.
