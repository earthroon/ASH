## 1. Purpose

`ASH-TCU-L16D` proves that one logical `16x16` software matrix tile can accumulate `K = 32` through `k_panel_count = 4` panels of width `8`, execute through native WGPU dispatch, and read back `1024` raw bytes equal to `256 f32` values.

L16D depends on L16A coordinate contract, L16B CPU mirror parity, and L16C single native logical16 readback parity.

This patch opens only KPanel4 on a single logical16 macro. It does not open K6B 4x4 macro grid dispatch, descriptor_count 16, production dispatch, runtime splice, or performance selection.
