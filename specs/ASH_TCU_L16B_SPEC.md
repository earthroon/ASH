## 1. Purpose

`ASH-TCU-L16B` validates one logical `16x16` software matrix tile by using the `ASH-TCU-L16A` subtile coordinate contract.

The patch proves that a CPU mirror arranged as four `8x8` output subtiles in fixed order `[C00, C01, C10, C11]` matches a CPU reference GEMM for the same logical `16x16` tile.

This patch is CPU-only. It does not open GPU execution, GPU readback, KPanel4 accumulation, runtime replacement, or performance selection.

## 5. Fixture SSOT

```text
fixture_id = deterministic_logical16_cpu_mirror_v1
m = 16
n = 16
k = 16
a_shape = [16,16]
b_shape = [16,16]
c_shape = [16,16]
a_scalar_count = 256
b_scalar_count = 256
c_scalar_count = 256
c_byte_len_f32 = 1024
k_panel_count = 1
```

Deterministic input formulas:

```text
A[row, kk] = ((row * 17 + kk * 13 + 3) % 31) / 31.0
B[kk, col] = ((kk * 19 + col * 7 + 5) % 37) / 37.0
```

## 6. CPU Reference Contract

```text
for row in 0..16:
  for col in 0..16:
    C_ref[row, col] = sum(A[row, kk] * B[kk, col] for kk in 0..16)
```

```text
reference_order = logical_row_major
reference_scalar_count = 256
reference_byte_len_f32 = 1024
```

## 7. CPU Mirror Contract

```text
cpu_mirror_output_order = subtile_contiguous
subtile_order = [C00, C01, C10, C11]
subtile_scalar_count = 64
logical_scalar_count = 256
logical_byte_len_f32 = 1024
```

Subtile output ranges:

```text
C00 = output[0..64]
C01 = output[64..128]
C10 = output[128..192]
C11 = output[192..256]
```

Mirror mapping:

```text
coord = logical16_coord_to_subtile([logical_row, logical_col])
local_index = coord.local_row * 8 + coord.local_col
output_index = coord.subtile_index * 64 + local_index
```

Comparison normalization:

```text
ref_index = logical_row * 16 + logical_col
mirror_index = subtile_index * 64 + local_row * 8 + local_col
compare C_ref[ref_index] vs C_mirror[mirror_index]
```
